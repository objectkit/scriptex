    PluginTemplate

    PluginTemplate is an strongly general purpose implementation suitable for prototyping lightweight MIDI applications.

    Constructor

    new PluginTemplate()

    See:

    • Plugin

    • Scriptex

    • Scripter

    • onMIdi

    • onParam

    • midi

    • deploy

    Example
    class Demonstrator extends PluginTemplate {
    // define Scipter.NeedsTimingInfo
    get needsTiming () {
       return true
    }

    // define Scripter.ResetParameterDefaults
    get needsDefaults () {
       return true
    }

    // define Scripter.PluginParameters
    get params () {
      return [
        {
          ID: "noteOffset"
        , type: "menu"
        , name: "Note Offset"
        , defaultValue: 0
        , valueStrings: [
             "1/4", "1/8", "1/16", "1/32"
          ]
        }
      ]
    }

    // exploit "ModelView" functionality by intercepting property changes
    set noteOffset (noteOffset) {
      this.beatsDelay = 1 / (noteOffset.splice("/").pop())
      // exploit the Scripter.MIDI object
      this.midi.allNotesOff()
    }

    // exploit MIDI event filtering
    onNote (noteOnOrOff) {
      // easily modify events with parameter values
      noteOnOrOff.beatPos = this.beatsDelay
      // exploit the functionality of #sendMidi
      return this.sendMidi(noteOnOrOff)
    }
  }

  // Easily deploy the plugin in the Scripter environment
  Demonstrator.deploy()
    .forEach(Trace) // [ HandleMIDI, PluginParameters, NeedsTimingInfo, ResetParameterDefaults ]

    onParam(key, val) → {number}

    Manage Scripter.ParameterChanged events by treating the plugin instance as a virtual "ViewModel".

    Given that ui parameter data has changed When the ui parameter has an ID Then plugin[ID] is assigned that ui parameter data

    This provides the interesting opportunity to intercept changes to UI by defining setters

    See:

    • ParameterChanged

    Parameters:
    Name Type Description
    key number

    a parameter index
    val number

    a parameter value
    Returns:
    Type:
    number
    Example
    class MidiStop extends PluginTemplate {
    get params () {
      return [
      , {
          ID: "doStopMidiNotes"
        , type: "momentary"
        , name: "MIDI Stop"
        }
      ]
    }

    // intercept the button press
    set doStopMidiNotes (pressed) {
      this.midi.allNotesStop()
    }
  }

    sendMidi(midi) → {number}

    Send a MIDI event using any one of the four available Scripter MIDI send methods.

    This general purpose solution imposes a small ruleset:

    • When beatPos is a string
      • Then beatPos is cast to number and sendAfterMilliseconds is invoked
    • When beatPos is a negative number
      • Then beatPos is inverted and SendMIDIEventAfterBeats is invoked
    • When beatPos is a positive integer
      • Then SendMIDIEventAtBeat is invoked
    • When neatPos is empty
      • Then SendMIDIEventNow is invoked
    See:

    • Scripter.SendMIDIEventNow

    • Scripter.SendMIDIEventAtBeat

    • Scripter.SendMIDIEventAfterBeats

    • Scripter.SendMIDIEventAfterMilliseconds

    Parameters:
    Name Type Description
    midi Event

    A Scripter MIDI event.
    Returns:
    Type:
